Database Connection Setup

This document provides the standard Mongoose database connection setup for the Shopez backend based on the required specifications. It ensures the application successfully connects to MongoDB before executing further logic.

Connection Code (config/db.js)

const mongoose = require('mongoose');

const connectDB = async () => {
    try {
        // Establish connection using the URI from environment variables
        await mongoose.connect(process.env.MONGO_URI);
        console.log("MongoDB Connected successfully");
    } catch (error) {
        // Log the error and exit the process with failure
        console.error(`Error: ${error.message}`);
        process.exit(1);
    }
};

module.exports = connectDB;

Code Breakdown

  • require('mongoose'): Imports the Mongoose library, which acts as the Object Data Modeling (ODM) interface for MongoDB.
  • connectDB function: An asynchronous wrapper function that manages the connection attempt.
  • process.env.MONGO_URI: Securely fetches the connection string from your environment variables (usually defined in a .env file) rather than hardcoding credentials.
  • Error Handling: The try...catch block ensures that if the connection fails, the error is logged to the console, and process.exit(1) shuts down the Node server (since an e-commerce app cannot function without its database).
